---
description: Break any problem down to fundamental truths, then rebuild solutions from atoms up
---
1. Clarify the problem with the user (1-2 questions max).
2. Phase 1 - Identify Assumptions: Ask what is assumed to be true that might not be. List them.
3. Phase 2 - Break to Atoms: For each assumption, ask for the most fundamental truth, asking "why?" until hitting bedrock facts.
4. Phase 3 - Rebuild From Truth: Start ONLY from verified fundamentals and ask for the simplest solution addressing the core need.
5. Provide the Output: Show the Problem, Assumptions Identified, Fundamental Truths, Rebuilt Solution, and contrast it vs Conventional approaches.
